Guide for Building Boost

Introduction

    As of JTSDK 3.2.2 Boost should be built with the mingw64 environment.

    Note that this is NOT the technique that the JTSDK64-Tools command Deploy-Boost uses.

    These steps will demonstrate how to deploy a Windows MME-Interface-based driver.

    If you want ALSA support – which requires a licensed SDK - refer to notes in the WSJT-X source.

Step 1: Download and fully deploy the JTSDK 3.2.2

  • The JTSDK Version 3.2.2 and any Tools packages can be found at: https://sourceforge.net/projects/hamlib-sdk/files/Windows/JTSDK-3.2-Stream/
  • Refer to the Deployment Guide for deploying JTSDK 3.2.2.

Step 2: Deploy the Build Tools

    The first stage involves deploying tools as if one were building Hamlib. Many required MSYS2 tools are deployed as part of this process. 

    • Execute the following command inside the JTSDK-Tools environment:
      • msys2
    • Execute the following commands inside the MSYS2 environment:
      • menu
    • Select the following menu options:
      • Select Option 2. to update the MSYS2 environment. If the environment requests that you close the window after completion then do so. Restart this section again.
      • Select Option 3. Install Hamlib dependencies.
      • Select Option 4. Install msys64 GNU Compilers.

    This should have now downloaded the build environment for mingw64 – providing a POSIX-compliant environment in which Boost code should be built.

    • Close all MSYS2 and JTSDK-Tools environment Windows

Step 3: Deploy the mingw64 Build Environment

    Launch an environment with the native GNU compilers.

    • Launch the JTSDK64-Tools environment
    • Inside the JTSDK64-Tools environment launch the mingw64 environment:
      • mingw64

Step 4: Download the Source

    You need to determine the download URL of the latest stable Boost release by clicking on the “Current Release” link at https://www.boost.org/ . At the time of writing the current version was 1.80.0.

    • Type the following commands:
      • cd ~/src
    • Right-click on the URL of the download file that you want – preferably the Unix .tar.gz file at https://www.boost.organd select “Copy Link”
    • In the msys64 shell, type wget, paste the URL, and then press enter. i.e.
      • wget https://boostorg.jfrog.io/artifactory/main/release/1.80.0/source/boost_1_80_0.tar.gz

    This downloads the source package

    • Extract the source package with:
      • tar xvfz boost_1_80_0.tar.gz

Step 5: Configure and build the Boost libraries

    The following is based off https://www.boost.org/doc/libs/1_63_0/more/getting_started/windows.html

    • Execute the following commands at the MSYS2 command prompt:
      • cd boost_1_80_0
      • ./bootstrap.sh
      • b2 install --prefix=/tmp/boost

    The above command places all that is built into the subdirectory /tmp/boost. This way the built libraries are easy to find.

Complete

    You will find a structure inside /tmp (Windows path x:\msys64\tools\msys64\tmp\boost) that contains your built Boost library set that can be inserted into the JTSDK.

    You will need to rename that folder to match the current version (i.e. rename folder  boost to 1.80.0) before it can be used with the JTSDK. 

    Deliver that folder structure to X:\JTSDK64-Tools\tools\boost\ .

    Make the appropriate adjustment in x:\JTSDK64-Tools\config\Versions.ini so that it can be used by the JTSDK.
