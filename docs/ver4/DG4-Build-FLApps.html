Guide for Building FL-Applications

    This site is designed to cement and promote techniques that work for building x64 versions of the FL-Apps with the Fast-Light Toolkit under the JTSDK64 MSYS2 Framework.

    • The term JT-ware is used to describe WSJT-X and any software forked and/or derived from WSJT-X.
    • The term FL-apps is used to describe any of the applications supported by David W1HKJ.

    At the time of writing only flrig (perhaps the most important tool for JT-Ware) can be compliled using these techniques. The MSYS2 and mingw64 environments are constantly evolving. Hopefully other apps will be available soon.

Step 1: Deploy the FL-Toolkit under the MSYS2/MinGW Environment

    From the JTSDK64-Tools Powershell Terminal: 

    • Type mingw64 to load the MSYS2/MinGW Development Environment
    • Type: menu
    •  From the "menu", select Option 5. Install FL-App dependencies.
    •  Close the MSYS2 terminal WIndow.
    •  Type mingw64 to re-load the MSYS2/MinGW Development Environment

Step 2: Download and Decompress the flrig source

    Note: These steps are valid for flrig 2.0.0.5. Should later versions become available (see: sourceforge.net/projects/fldigi/files/flrig/ ) adjust instructions accordingly.    

    Under the MSYS2/MinGW environment:

    • Create a directory ~/src (if it does not already exist) and change into that directory:
      • mkdir ~/src
      • cd ~/src
    • Execute the following command to retrieve flrig 2.0.0.5:
      • wget https://sourceforge.net/projects/fldigi/files/flrig/flrig-2.0.05.tar.gz
    • Extract the source from the downloaded package with:
      • tar xvfz flrig-2.0.05.tar.gz

    This should have now downloaded and decompressed the source file package. Change into the build directory:

    • cd flrig-2.0.05

Step 3: Compile the software

    Execute the following commands:

    • ./configure --enable-static CFLAGS="-static" LDFLAGS="-static" CXXFLAGS="-static"
    • make nsisinst

Complete

    You should have a readily built flrig-2.0.05_setup.exe .exe installer inside the source tree i.e. ~/src/flrig-2.0.05/src .
